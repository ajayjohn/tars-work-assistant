# Generation Reference

Use for naming, positioning, campaign angles, product ideas, and non-obvious
options.

1. Generate several distinct frames, not variations of one frame.
2. Use subagents only for separable lenses; each returns bounded JSON with
   `frame`, `ideas`, `why_it_might_work`, and `risks`.
3. Cluster ideas by strategic usefulness, not cleverness alone.
4. Mark which ideas are immediately usable, which need validation, and which
   are deliberately speculative.
5. Offer one next move that requires minimal user time.

Do not save ideas unless the user asks to persist them.
